1
Mulai Pertama Kali
Ikuti urutan ini agar semua fitur terhubung dengan benar.
1
Taruh semua file dalam 1 folder yang sama — index.html, toko.html, kasir.html, inventory.html, laporan.html, pelanggan.html, cara-pakai.html harus satu folder.
2
Buka toko.html di browser — isi profil toko, upload logo, setup struk. Ini adalah data dasar yang dipakai semua halaman lain.
3
Setup password di toko.html → tab Profil → card 🔐 Password & Akses. Isi password Owner, Admin, dan Kasir.
4
Input produk di inventory.html — tambahkan semua produk beserta harga jual, harga modal, dan stok awal.
5
Buka index.html — login sebagai Owner, semua app sudah siap digunakan.
💡 Isi profil toko sedetail mungkin — nama, alamat, telepon, dan logo akan muncul otomatis di struk, laporan, dan header semua halaman.
2
Sistem Login & Akses
TokoPOS memiliki 3 level akses untuk membatasi siapa yang bisa melihat dan mengubah data.
3 Role Pengguna
👑 OWNER Akses penuh ke semua halaman
Bisa melihat laporan keuangan, mengubah pengaturan toko, setup password, dan melakukan semua yang bisa dilakukan Admin & Kasir.
📋 ADMIN Input & kelola data
Bisa akses Inventory (tambah/edit/hapus produk), Pelanggan (tambah/edit/hapus), dan Kasir. Tidak bisa lihat Laporan Keuangan atau ubah Pengaturan Toko.
🏷️ KASIR Transaksi saja
Hanya bisa akses halaman Kasir. Riwayat transaksi hanya tampil untuk hari ini saja. Tidak bisa hapus riwayat, tidak bisa akses halaman lain.

Setup Password
1
Buka toko.html → tab Profil Toko → scroll ke bawah → card 🔐 Password & Akses
2
Isi Password Owner (wajib) — minimal 4 karakter, bisa angka/huruf/kombinasi. Contoh: toko2024 atau 1234
3
Isi Password Admin + nama admin (opsional). Jika dikosongkan, role Admin tidak aktif.
4
Isi Password Kasir + nama kasir (opsional). Semua password harus berbeda satu sama lain.
5
Klik 💾 Simpan Password. Mulai sekarang setiap buka index.html akan muncul form login.
Flow Login Sehari-hari
  • Buka index.html → pilih role → masukkan password → tekan Enter atau klik Masuk
  • Login Owner / Admin → masuk ke dashboard, bisa navigasi semua app
  • Login Kasir → langsung redirect ke kasir.html
  • Klik Keluar di topbar manapun untuk logout
  • Session aktif selama tab browser tidak ditutup. Buka tab baru = perlu login ulang
⚠️ Catat password Owner di tempat yang aman. Jika lupa, bisa dihapus lewat toko.html → Password & Akses → Hapus Password (selama masih login sebagai Owner).
3
⚙️ Pengaturan Toko
File toko.html adalah pusat konfigurasi. Buka file ini pertama kali sebelum menggunakan yang lain. 👑 Owner only
Tab Profil Toko
  • Isi nama toko, jenis usaha, telepon, email, alamat, kota, website/sosmed
  • Upload logo (JPG/PNG) — muncul di struk, laporan, dan halaman login
  • Pilih warna aksen tema aplikasi
  • Isi modal awal dan kas awal untuk perhitungan laporan
Tab Struk & Cetak
  • Atur teks header struk baris 1 & 2 (default: nama toko & tagline)
  • Atur pesan footer struk (contoh: "Terima kasih sudah berbelanja!")
  • Toggle tampilkan/sembunyikan nomor referensi transaksi
  • Preview tampilan struk secara langsung di sebelah kanan
Tab Backup & Restore
  • Download Backup JSON — simpan semua data ke file (produk, transaksi, pelanggan)
  • Download CSV — export data ke format Excel
  • Restore — pulihkan data dari file JSON backup sebelumnya
  • 5 slot auto-backup tersimpan otomatis setiap ada perubahan
⚠️ Lakukan backup rutin! Data tersimpan di browser — jika cache dihapus atau ganti browser/HP, data bisa hilang.
Tab Data & Reset
  • Lihat statistik: jumlah produk, transaksi, pelanggan
  • Reset data secara selektif: hanya transaksi, hanya stok, atau semua
🔐 Password & Akses Owner Only
  • Setup password untuk 3 role: Owner, Admin, Kasir
  • Password bebas — angka, huruf, atau kombinasi, minimal 4 karakter
  • Semua password harus berbeda satu sama lain
  • Hapus semua password untuk menonaktifkan sistem login

📦 Inventory & Stok
Kelola semua produk yang dijual. Data produk terhubung langsung ke Kasir. 👑 Owner 📋 Admin
Tab Stok
  • Lihat semua produk dalam bentuk kartu dengan info stok
  • Produk dengan stok di bawah minimum tampil dengan badge merah otomatis
  • Filter berdasarkan kategori atau cari dengan kolom pencarian
  • Klik kartu produk untuk lihat detail, edit, atau hapus

Tab Mutasi Stok
  • Catat penambahan stok (barang masuk dari supplier)
  • Catat pengurangan stok manual (rusak, hilang, sampel)
  • Setiap mutasi tercatat dengan tanggal dan keterangan
  • Stok otomatis berkurang setiap kali ada transaksi di kasir
Tab Label Barcode
  • Pilih produk dan cetak label harga + barcode
  • Pilih ukuran label (kecil, sedang, besar) dan jumlah cetak
  • Barcode yang tercetak bisa langsung discan di kasir.html
💡 Klik tombol ↻ Sync di kasir.html untuk memuat ulang data produk terbaru dari Inventory.
5
🛒 Kasir & Transaksi
Halaman utama untuk melayani pembeli. 👑 Owner 📋 Admin 🏷️ Kasir
Mencari Produk
  • Ketik nama produk atau SKU di kolom scan atas untuk filter cepat
  • Klik ikon 📷 untuk scan barcode lewat kamera HP atau laptop
  • Filter produk berdasarkan kategori dengan tombol pill di bawah scan bar
  • Klik kartu produk untuk langsung menambahkan ke keranjang
💡 Fitur scan barcode bekerja paling baik di Chrome atau Edge. Izinkan akses kamera saat diminta. Gunakan BarcodeDetector API yang tersedia di Chrome 83+.
Proses Transaksi
1
Klik produk atau scan barcode → produk masuk keranjang
2
Atur jumlah dengan tombol + dan −
3
Isi diskon jika ada — bisa dalam % atau nominal Rp. Tombol cepat 10% dan 20% tersedia.
4
Pilih metode bayar: 💵 Tunai atau 📲 QRIS
5
Jika Tunai, isi jumlah uang diterima — kembalian dihitung otomatis. Klik nominal cepat yang tersedia.
6
Klik 💳 Proses Pembayaran → struk muncul
Setelah Transaksi — Di Struk
  • Pilih Pelanggan dari dropdown agar transaksi tercatat di riwayat pelanggan & segmen otomatis naik
  • 🖨️ Print — cetak struk ke printer thermal atau kertas biasa
  • 📲 Kirim WA — jika pelanggan dipilih & punya nomor WA, langsung buka chat. Jika tidak, teks disalin ke clipboard
  • ➕ Transaksi Baru — reset keranjang untuk transaksi berikutnya
Riwayat Transaksi
  • Klik tombol 📋 Riwayat di topbar untuk membuka panel riwayat
  • 👑 📋 Bisa lihat semua riwayat + hapus riwayat
  • 🏷️ Hanya lihat transaksi hari ini saja, tidak bisa hapus
  • Hapus riwayat tidak mempengaruhi data laporan keuangan yang sudah tercatat
6
📊 Laporan Keuangan
Laporan keuangan lengkap berbasis data transaksi kasir dan stok inventory. 👑 Owner only
Tab-tab yang Tersedia
  • Dashboard — grafik penjualan harian/mingguan, ringkasan omset, laba, jumlah transaksi, produk terlaris
  • Jurnal Umum — semua transaksi format double-entry, filter tanggal, export CSV
  • Buku Besar — ringkasan per akun: saldo awal, mutasi, saldo akhir
  • Laba Rugi — laporan P&L otomatis, pilih periode bulanan/kuartalan/custom
  • Neraca — balance sheet sederhana: Aset, Liabilitas, Ekuitas
  • Arus Kas — cash flow dari aktivitas operasional berdasarkan tanggal aktual
  • Penjualan — detail semua transaksi dengan filter tanggal & produk, export CSV
💡 Semua tab laporan bisa di-export ke CSV yang bisa langsung dibuka di Microsoft Excel atau Google Sheets untuk analisis lebih lanjut.
7
👥 Pelanggan & CRM
Kelola hubungan dengan pelanggan: simpan data, kirim promo, ingatkan ulang tahun. 👑 Owner 📋 Admin
Daftar Pelanggan
  • Tabel semua pelanggan: nama, nomor WA, segmen, total belanja, ulang tahun
  • Filter berdasarkan segmen: New, Regular, VIP
  • Klik nama pelanggan untuk buka panel detail di sisi kanan
  • Segmen naik otomatis berdasarkan total belanja & jumlah transaksi via kasir
Segmentasi Otomatis
🆕 NewPelanggan baru, belum ada riwayat transaksi
⭐ RegularTotal belanja ≥ Rp 200.000 atau sudah 3+ transaksi
👑 VIPTotal belanja ≥ Rp 1.000.000 atau sudah 10+ transaksi
Blast Promo WA
  • Buka tab Blast Promo → tulis pesan (gunakan {nama} untuk nama otomatis)
  • Filter tujuan berdasarkan segmen (semua, VIP, Regular, New)
  • Centang pelanggan yang akan dikirimi, atau klik ☑️ Pilih Semua
  • Klik 📲 Kirim — WhatsApp terbuka satu per satu ke setiap nomor
  • Status ✅ Terkirim tercatat per pelanggan dalam sesi tersebut
Reminder Ulang Tahun
  • Buka tab Ulang Tahun — tampil pelanggan yang ulang tahun hari ini & 7 hari ke depan
  • Centang pelanggan yang akan dikirimi ucapan
  • Klik 📲 Kirim — pesan ucapan otomatis dikirim via WA
  • Notif ulang tahun juga muncul di kasir.html setiap kali halaman dibuka

💡 Tips & FAQ
Pertanyaan yang sering ditanyakan dan tips penggunaan terbaik.
Backup yang Disarankan
  • Setiap minggu — download backup JSON dari toko.html → Tab Backup & Restore
  • Setiap bulan — download laporan CSV dari laporan.html
  • Sebelum hapus cache browser — selalu backup dulu!
  • Simpan file backup di Google Drive atau folder cloud agar aman
FAQ
Di mana data tersimpan?›
Data tersimpan di localStorage browser di komputer/HP yang digunakan. Data tidak dikirim ke server manapun. Karena itu, data hanya ada di browser tersebut — tidak otomatis sinkron ke perangkat lain.
Apakah bisa digunakan offline?›
Ya, semua file bisa digunakan tanpa koneksi internet. Internet hanya dibutuhkan untuk memuat font dari Google Fonts saat pertama kali dibuka — setelah itu font ter-cache di browser.
Apakah bisa dibuka di HP?›
Ya, semua file sudah responsif untuk HP. Namun untuk kasir dan laporan, layar yang lebih besar (tablet atau laptop) lebih nyaman karena banyak data yang ditampilkan secara bersamaan.
Data hilang setelah hapus cache browser?›
Ya, data bisa hilang jika cache/storage browser dihapus. Solusinya: lakukan backup rutin lewat toko.html → Tab Backup & Restore → Download Backup JSON sebelum hapus cache.
Bisa digunakan di beberapa komputer?›
Data tidak sinkron antar perangkat secara otomatis. Untuk pindah data, export backup JSON di satu komputer lalu import/restore di komputer lain lewat toko.html. Data akan terbaca di semua perangkat yang melakukan restore.
Scan barcode tidak berfungsi?›
Pastikan menggunakan Chrome atau Edge versi terbaru. Izinkan akses kamera saat diminta browser. Fitur ini menggunakan BarcodeDetector API yang belum tersedia di Firefox atau Safari. Jika tidak bisa, gunakan input manual atau barcode scanner USB.
Lupa password Owner?›
Jika masih ada sesi login yang aktif: buka toko.html → Password & Akses → Hapus Password. Jika sudah logout semua: buka browser developer tools (F12) → Application → Local Storage → hapus key toko_auth_config. Semua password akan hilang dan akses terbuka kembali.
Kirim WA harus pilih kontak satu-satu?›
Di kasir: jika pelanggan dipilih dari dropdown dan punya nomor WA tersimpan, WA langsung membuka chat ke nomor tersebut. Di pelanggan/blast promo: WA terbuka satu per satu ke setiap nomor yang dicentang — ini adalah limitasi dari WhatsApp, bukan dari aplikasi.
